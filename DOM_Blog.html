What is Document Object Model (DOM) ?

It is a programming interface for HTML and XML documents. DOM is created by the browser when a webpage is loaded and creates a model of that page. This model is the Tree.The Document Object Model (DOM) is a tree-like representation of webpage contents that uses nodes instead of elements. HTML elements are represented by these nodes, which can be manipulated and re-rendered.
JavaScript DOM is the API that JavaScript being used to communicate to static HTML elements, making them more functional.
In simpler terms, Its Models the entire HTML document in an object to access the properties.

What are DOM elements ?

    1. Root Node - The root node is the <Doctype html>
    2. Parent Node - The Parent node is the node with a child or children.
    3. Child Node - Any subnode of a given parent node is called a child node
    4. Siblings Node - All the children Nodes of same parent Nodes are called siblings Node


What actions can we perform with this DOM model in JavaScript ?

    - Change/Remove HTML elements in the DOM/on the page.
    - Change and add CSS styles to elements.
    - Create new elements and insert them into the DOM/page.
    - Attach elements with event listeners (click, keypress, submit).

Some DOM Manipulations Methods : -

    1. getElementById() - returns an Element object representing the element whose id property matches the specified string.

    2. getElementByClassName() - returns an array-like object of all child elements whose ClassName property matches the specified string.

    3. getElementByTagName() - returns an returns a list of live HTML elements with the give TagName.

    4. querySelector() - returns the first element that matches one or more CSS selectors. If no match is found, it returns null.

    3. querySelectorAll() - returns “nodelist” containing references to all of the matches of the CSS selectors. If no match is found, it returns null.
Apart from the methods mentioned here, W3C specification supports over a hundred DOM styles, methods, events, documents, and event objects
